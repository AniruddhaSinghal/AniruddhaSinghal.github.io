Section 04 · Cross-Modal Fusion

Bridging vision and language

The transformer's cross-attention lets every language token "look at" the entire image. Click any token to see exactly which scene regions it attends to — and why.

What is cross-attention?
In cross-attention, Queries Q come from language tokens — each word asks "which image regions are relevant to me?" The Keys K and Values V come from image patches — each patch answers "here is what I contain."
CrossAttn(Q,K,V) = softmax( Q·Kᵀ / √d_k ) · V
Why this matters: "red" learns to attend to red-coloured patches. "cube" attends to box-shaped regions. "shelf" attends to the upper workspace. This grounding is what makes VLA work.

Stage-by-Stage Detail
STAGE 0
Scene Captured
The robot's camera captures a raw RGB image of the workspace. This image (typically 224×224 or 256×256 pixels) is the sole visual input to the VLA model.
STAGE 1
Visual Tokenization (ViT)
The image is split into 16×16 patches (e.g. 14×14 = 196 patches for a 224px image). Each patch is linearly projected to a d-dimensional embedding — the same dimension as text tokens.
STAGE 2
Language Tokenization
The text instruction is tokenized using a BPE (Byte-Pair Encoding) vocabulary. Each token becomes a learned embedding vector. Positional encoding is added to preserve word order.
STAGE 3
Transformer Cross-Attention
Image and language tokens are concatenated and processed jointly through transformer layers. Cross-attention allows language tokens to ground themselves in specific visual regions and vice versa.
STAGE 4
Action Token Decoding
The fused representation is passed to an action head. Action tokens are decoded auto-regressively — each step produces a Δ(joint angle) or gripper command. The full trajectory is T timesteps.
STAGE 5
Execution + Feedback
Actions are sent to the robot controller. At each step, the camera captures a new frame and the model re-runs inference — this closed-loop control allows real-time correction.
Key VLA Properties
① No explicit symbolic planning step
② End-to-end differentiable
③ Language generalises to unseen tasks
④ Vision grounds language to the real world
⑤ Actions are continuous, not discrete
⑥ Closed-loop: re-plans every frame
