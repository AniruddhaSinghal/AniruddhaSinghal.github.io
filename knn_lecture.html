CHAPTER 2
Section 02 · Core Idea

How KNN thinks

KNN doesn't learn a model — it memorises the data. Classification is entirely based on proximity at prediction time.

The Fundamental Idea
KNN rests on a beautifully simple assumption: things that are close to each other tend to belong to the same category.

To classify a new point, we ask: "Who are your K nearest neighbours? What are they?" — and we let the majority vote decide.
Class(x) = majority vote of K nearest training points
Algorithm Steps
STEP 01
Store all training data
Unlike most algorithms, KNN has no "training" phase. It simply stores all labelled data points in memory. This is why KNN is called a lazy learner.
STEP 02
Receive a new query point
A new, unlabelled point arrives. We need to decide which class it belongs to. We have no prior model — only the stored training data.
STEP 03
Compute distances to all points
Calculate the distance from the query point to every single training point. Most commonly: Euclidean distance d = √((x₂-x₁)² + (y₂-y₁)²)
STEP 04
Select the K nearest neighbours
Sort all training points by distance. Select the top K closest ones. These become the "voters" for classification. K is a hyperparameter you choose.
STEP 05
Majority vote → assign class
Count how many of the K neighbours belong to each class. The class with the most votes wins. Ties are broken by choosing the closest neighbour's class.

What does K control?
K is the only knob in KNN. It controls the smoothness of the decision boundary:
Small K (1–3)
Large K (11–21)
Very jagged boundary. Fits training data perfectly. Likely to overfit — noise becomes pattern.
Smooth boundary. Ignores individual anomalies. May underfit if K is too large.
CHAPTER 5
Section 05 · Hyperparameter Tuning

The art of choosing K

K is a hyperparameter — it's not learned from data, it's chosen by you. Too small and you overfit. Too large and you lose precision.

Accuracy vs K — training vs test
The sweet spot: Look for the K where test accuracy is highest. This is typically where the model generalises best — not memorising noise (small K) nor being too coarse (large K).

Golden Rules
① Always normalise / scale your features
② Try odd K values to avoid ties
③ Use cross-validation to pick K
④ √n is a common starting point for K
⑤ Fewer dimensions → KNN works better
